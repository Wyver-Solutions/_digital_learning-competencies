Framework Structure

The framework is designed around six competency areas, each divided into five sub-categories.

Within each sub-category there are three level descriptors:

  • Level 1: Foundation
  • Level 2: Practioner
  • Level 3: Advanced

Each level descriptor says what a person should be able to do who is working at that level in that area.

Descriptors use Bloom's taxonomy of verbs. See: https://tips.uark.edu/using-blooms-taxonomy/

For developers

The competency framework is held within a git repository.

The repository contains a single YAML file describing each competency area.

Within the top section of the file (between the three dashes: --- ) you will find the data for the framework:


  ---
  title: The competency area
  ID: The ID code for the competency area
  description: The description of the competency area
  items: The list of sub-categories
  - ID: The ID code for the sub-category
    name: The name of the sub-category
    L1: The descriptor for level 1
    L2: The descriptor for level 2
    L3: The descriptor for level 3
  - ID: The next sub-category ID code
    name: etc
  ---

The files are designed primarily to work as a Jekyll collection (Docs), but can also be used in any system which can iterate over files and over the YAML data contained within them.

To incorporate the competencies into your project, you can add them as a git submodule:

git submodule add https://github.com/berthelemy/_digital-learning-competencies _digital-learning-competencies

The underscore is so the files work directly as a Jekyll collection.

You will need to run the above command inside your Collections directory, and then use git pull inside the submodule whenever you want to update.

There is also a machine readable json file.

This is an ALPHA version. Please do not use for any production service.
